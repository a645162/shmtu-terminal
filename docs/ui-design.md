# 上海海事大学终端应用 — 界面设计文档

> 版本：1.0 | 更新日期：2026-05-22

---

## 一、整体导航结构

### Avalonia 端

Avalonia 作为桌面原生应用，不受单页面限制，采用**多窗口**架构：

```
┌─────────────────────────────────────────────┐
│                   主窗口                      │
│  ┌────────┬────────┬──────────┐             │
│  │ 首页Tab │ 账单Tab │ 功能大全Tab │             │
│  └────────┴────────┴──────────┘             │
│  [Tab 内容区]                                │
└─────────────────────────────────────────────┘

独立窗口：
  - 设置窗口
  - 关于窗口
  - 身份/账号管理窗口
  - 数据导入导出窗口
  - 验证码测试窗口
  - 启动密码窗口（模态）
  - 身份选择窗口（模态）
```

### Tauri 端

Tauri 采用**单页面多路由**架构，对应功能以页面或模态对话框实现：

```
┌─────────────────────────────────────────────┐
│                   主页面                      │
│  ┌────────┬────────┬──────────┐             │
│  │ 首页Tab │ 账单Tab │ 功能大全Tab │             │
│  └────────┴────────┴──────────┘             │
│  [Tab 内容区]                                │
└─────────────────────────────────────────────┘

对话框/路由：
  - 设置对话框
  - 关于对话框
  - 身份/账号管理对话框
  - 数据导入导出对话框
  - 验证码测试对话框
  - 启动密码对话框（模态）
  - 身份选择对话框（模态）
```

## 二、页面布局设计

### 2.1 主窗口

主窗口采用顶部 Tab 导航，分为三个 Tab。

```
┌──────────────────────────────────────────────────────┐
│  海大终端                                    — □ ✕   │
├──────────────────────────────────────────────────────┤
│  [ 首页 ]  [ 账单 ]  [ 功能大全 ]                      │
├──────────────────────────────────────────────────────┤
│                                                      │
│                   (Tab 内容区)                         │
│                                                      │
│                                                      │
│                                                      │
│                                                      │
├──────────────────────────────────────────────────────┤
│  状态栏：当前身份 | 同步状态 | 最后同步时间              │
└──────────────────────────────────────────────────────┘
```

**窗口尺寸**：默认 1000×650，最小 800×500

**主窗口工具栏/菜单**（Avalonia 右键菜单支持）：
- 文件：数据导入/导出
- 编辑：身份管理、账号管理
- 工具：验证码测试、同步数据
- 帮助：设置、关于

## 三、关键交互流程

### 3.1 应用启动流程

```
启动应用
    │
    ▼
检查启动保护是否开启
    │
    ├─ 是 → 弹出密码窗口 → 密码正确? → 否 → 重新输入
    │                           │
    │                           ↓ 是
    │
    └─ 否
    │
    ▼
检查是否记住默认身份
    │
    ├─ 是 → 检查默认身份是否存在
    │         │
    │         ├─ 存在 → 直接进入主窗口（使用该身份）
    │         └─ 不存在 → 弹出身份选择窗口
    │
    └─ 否 → 弹出身份选择窗口
                │
                ▼
          用户选择身份 → 进入主窗口
```

### 3.2 账单同步流程

```
用户点击"同步"按钮
    │
    ▼
显示同步进度对话框
    │
    ▼
遍历当前身份下的启用账号
    │
    ▼
对每个账号:
    ├─ 尝试恢复已保存的 cookies
    │   ├─ cookies 有效 → 跳过登录
    │   └─ cookies 无效 → 执行 CAS 登录
    │       ├─ 获取验证码
    │       ├─ 根据配置选择识别方式
    │       ├─ 提交登录
    │       └─ 登录成功 → 保存 cookies
    │
    ├─ 增量同步: 逐页获取账单
    │   ├─ 解析页面
    │   ├─ 检查交易号是否已存在
    │   ├─ 新条目写入账号原始数据表
    │   └─ 达到提前停止阈值 → 停止
    │
    └─ 新条目追加到身份合并数据表
    │
    ▼
同步完成 → 更新界面 → 显示同步结果摘要
```

### 3.3 验证码处理流程

```
CAS 登录需要验证码
    │
    ▼
获取验证码图片
    │
    ▼
根据设置选择识别模式:
    │
    ├─ 手动输入 → 显示图片 → 等待用户输入
    │
    ├─ 远程 OCR → TCP 连接 → 发送图片 → 接收结果
    │              ├─ 成功 → 返回表达式/答案
    │              └─ 失败 → 自动重试(最多N次) → 仍失败则降级为手动
    │
    └─ 本地 ONNX → 加载模型(首次) → 图像预处理 → 推理 → 返回结果
                    └─ 模型不存在 → 自动下载 → 加载
    │
    ▼
结果为表达式? → 是 → 计算最终答案
              → 否 → 直接使用
    │
    ▼
提交验证码
```

### 3.4 数据管理流程

```
身份合并数据操作:
    │
    ├─ 手动添加记录 → 写入合并数据表 → 记录操作日志
    │
    ├─ 手动删除记录 → 从合并数据表删除 → 记录操作日志
    │
    ├─ 全量更新(账号级) → 覆盖账号原始数据表 → 同步到合并表 → 清空该账号操作日志
    │
    └─ 全量更新(身份级) → 对所有账号执行全量更新 → 清空该身份操作日志

数据导出:
    选择范围 → 选择格式 → 选择路径 → 生成文件

数据导入:
    选择文件 → 预览数据 → 确认导入 → 写入数据库

数据快照:
    创建 → 复制当前数据库 → 带时间戳存储
    恢复 → 选择快照 → 确认 → 替换当前数据库
    清理 → 超过保留数时删除最旧的快照
```

### 4.3 通用交互规范
